Core Philosophy

Quality Over Quantity: 50-100 clean, relevant records are better than 1,000 messy records.

Collect Once, Reuse Forever: Run your workflow once, save the data, then work with that saved data. This prevents API rate limits, bans, and wasted time re-collecting.

Data Source Tiers

Before you start, understand which data sources to use:

Tier 1: RECOMMENDED (Start Here)

  • Public Datasets: Kaggle, HuggingFace, Google Dataset Search, Data.gov, GitHub
  • RSS Feeds: No authentication, no rate limits, perfect for news aggregation
  • YouTube Data API: Generous free tier (10,000 quota units/day)
  • NewsAPI: 100 requests/day on free tier

Tier 2: Use With Caution

  • Twitter datasets from Kaggle: Archived data, not live API
  • Reddit archives: Pushshift datasets, NOT live scraping

Tier 3: AVOID

  • LinkedIn API: Complex approval, expensive, inaccessible for learners
  • Twitter/X API free tier: Severe rate limiting, gets banned easily
  • Reddit API live scraping: Known to ban IPs mid-project
  • Business verification APIs: Expensive, complex approval

Start with Tier 1 sources. You can always add complexity later.

Common Challenges & Solutions

"What data should I collect?"

  • Review your Assignment 2 problem
  • Search Kaggle for "[your topic] dataset"
  • Find 2-3 relevant RSS feeds
  • Aim for 100-200 total records

"APIs seem complicated"

  • Start with public datasets and RSS feeds
  • They require no authentication
  • Add APIs only if needed

"How much data is enough?"

  • Minimum: 50-100 records
  • Strong: 100-200 records
  • Excellence: 150-300 records
  • NOT 500+ or 1000+ (quality over quantity)

"My workflow seems too simple"

A 6-node workflow is perfectly acceptable:

  • Manual Trigger
  • 3 Data Sources (RSS/HTTP Request)
  • Function (clean and structure)
  • Write Binary File

Simple and working beats complex and broken.

Tips for Excellence

  • Use Tier 1 sources - Public datasets, RSS, YouTube API, NewsAPI
  • Collect once, reuse forever - Don't re-run your workflow repeatedly
  • Focus on quality - Clean, relevant data beats large messy datasets
  • Document thoroughly - Someone else should be able to run your workflow
  • Test 3+ times - Verify your workflow runs reliably

Examples of Strong Projects

  • Brand Voice Analysis: 180 records from YouTube API + Kaggle dataset, 95% quality rate
  • Industry News Aggregator: 156 records from RSS feeds + NewsAPI, 100% quality rate
  • Sentiment Analyzer: 200 labeled records from public datasets, 98.5% quality rate

What NOT to Do:

  • Trying to collect 2000+ tweets from free Twitter API (gets rate limited)
  • Using LinkedIn API without enterprise access (won't work)
  • Live scraping Reddit (gets IP banned)
  • Prioritizing quantity over quality
